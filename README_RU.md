# SiteMoney Audit

Готовый микро-продукт для быстрого заработка на аудитах сайтов малого бизнеса.

Что делает:
- сам ищет бизнесы по городам мира через OpenStreetMap/Overpass;
- забирает название, сайт, телефон, карту и базовые контакты бизнеса;
- автоматически аудирует найденные сайты и сортирует самые денежные цели;
- скачивает сайт и проверяет HTML-сигналы: CTA, контакты, формы, отзывы, локальные сигналы, schema, analytics, mobile viewport, HTTPS, скорость ответа;
- считает `Conversion score` и примерную утечку денег;
- строит продаваемые приоритеты правок;
- генерирует email, follow-up, DM и Telegram-сообщение;
- предлагает 3 пакета услуги;
- ведет локальную историю, CRM-следующее действие и экспорт HTML/JSON/CSV.

Важно: оценка денег нужна для продажи разговора. Она не гарантирует доход.

## Локальный запуск

На этом компьютере можно просто запустить:

```powershell
.\RUN_LOCAL.ps1
```

Открыть: `http://127.0.0.1:8787`

Если Node.js установлен обычным способом:

```powershell
npm install
npm run build
npm start
```

Открыть: `http://127.0.0.1:8787`

Проверка версии: `http://127.0.0.1:8787/__version`

## Быстрый способ продавать

1. Выбрать нишу: plumber, dentist, clinic, salon, lawyer, restaurant, hotel, cleaning, electrician и т.д.
2. Нажать `Найти бизнесы`.
3. Приложение само найдет бизнесы, сайты, телефоны, карты и запустит аудит лучших сайтов.
4. Открыть блок `Найденные бизнесы автопилотом`.
5. Скопировать готовое сообщение владельцу.
6. Продавать сначала `Quick Fix`, не большой редизайн.

Автоматизация ищет и готовит лиды сама. Финальную отправку сообщения лучше делать вручную, чтобы не нарушать правила площадок и не ловить блокировки аккаунтов.

## Деплой

Проект готов к Railway:

```powershell
npm install
npm run build
npm start
```

Railway использует `railway.json`, `Procfile`, `/health` и `/__version`.
Production entrypoint: `node app-server.mjs`.

Проверка после деплоя:

- `/__version` должен показать `SITEMONEY_AUDIT_20260622_RUNTIME_REBUILD`;
- `/__frontend-check` должен показать `"ok": true`;
- если frontend старый, сервер сам попробует пересобрать `dist` при запуске.

Папка `dist/` специально не игнорируется: она входит в deploy как готовый frontend fallback.

Если Railway показывает `ERROR packages field missing or empty`, проверь что в корне есть
`pnpm-workspace.yaml` с блоком:

```yaml
packages:
  - "."
```
